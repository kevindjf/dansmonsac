## Technical Details

### New Tables

#### 1. DailyChecks Table
Purpose: Track daily supply checks for bag preparation

| Column | Type | Description |
|--------|------|-------------|
| id | TEXT (PK) | Unique identifier |
| date | DATETIME | Date for which supplies are checked |
| supplyId | TEXT | Reference to supply |
| courseId | TEXT | Reference to course |
| isChecked | BOOLEAN | Whether supply is checked (default: false) |
| createdAt | DATETIME | Timestamp of creation |

**Operations:**
- `getDailyChecksByDate(DateTime)` - Get all checks for a specific date
- `getDailyCheckBySupply(String, DateTime)` - Get check for specific supply on date
- `insertDailyCheck(DailyChecksCompanion)` - Insert new check
- `updateDailyCheck(DailyChecksCompanion)` - Update existing check
- `deleteDailyCheck(String)` - Delete check by ID

#### 2. BagCompletions Table
Purpose: Record when bag preparation is completed (for streak calculation)

| Column | Type | Description |
|--------|------|-------------|
| id | TEXT (PK) | Unique identifier |
| date | DATETIME | Date for which bag was prepared |
| completedAt | DATETIME | Timestamp of completion |
| deviceId | TEXT | Device identifier (for syncing) |
| createdAt | DATETIME | Timestamp of creation |

**Operations:**
- `getAllBagCompletions()` - Get all completions
- `getBagCompletionByDate(DateTime)` - Get completion for specific date
- `getBagCompletionsInRange(DateTime, DateTime)` - Get completions in date range (for streak calculation)
- `insertBagCompletion(BagCompletionsCompanion)` - Insert new completion
- `deleteBagCompletion(String)` - Delete completion by ID

#### 3. PremiumStatus Table
Purpose: Store premium purchase and parent linking status (Epic 4 & 5)

| Column | Type | Description |
|--------|------|-------------|
| id | TEXT (PK) | Unique identifier |
| hasPurchased | BOOLEAN | Whether user purchased premium (default: false) |
| linkedParentId | TEXT (nullable) | Parent device ID if linked |
| updatedAt | DATETIME | Timestamp of last update |
| createdAt | DATETIME | Timestamp of creation |

**Operations:**
- `getPremiumStatus()` - Get current premium status
- `insertPremiumStatus(PremiumStatusCompanion)` - Insert new status
- `updatePremiumStatus(PremiumStatusCompanion)` - Update status
- `setPurchased(bool)` - Update purchased flag
- `setLinkedParent(String?)` - Update linked parent ID

### Migration Strategy

```dart
@override
MigrationStrategy get migration => MigrationStrategy(
  onCreate: (Migrator m) async {
    await m.createAll(); // Creates all tables including v3 tables
  },
  onUpgrade: (Migrator m, int from, int to) async {
    if (from < 2) {
      // Migration v1 → v2 (preserved for backwards compatibility)
      // ...
    }
    if (from < 3) {
      // Migration v2 → v3: Add new tables for Epic 2
      await m.createTable(dailyChecks);
      await m.createTable(bagCompletions);
      await m.createTable(premiumStatus);
    }
  },
);
```

**Key Points:**
- Non-destructive migration - only adds new tables
- Existing v1→v2 migration preserved
- All existing data (courses, supplies, calendar_courses) remains intact
- Migrations run automatically on app startup when version mismatch detected
